# algebraic-filter は何のために在るか

[English](philosophy.md)

> 理想は大きく、 現実は誠実に。 この文書は両方を保持する — AF が目指す大きな理想と、
> 今日それが保証できることの誠実で控えめな現実を。

## 一文の thesis

AI のコード生成は **確率論**である — もっともらしいコードをサンプルするだけで、 何も
証明しない。 algebraic-filter は **その確率の海から、 決定論的に保証された島を切り出す**:
hook を pass した任意のコードについて、 ある性質群が決定論ツールで *認証* される
(サンプリングなし・LLM なし・「たぶん」なし)。 その島の中の性質を、 人間はもうレビュー
しなくてよい。

**島を広げること = AI に、 自分で確認せず安全に任せられる範囲を広げること。** これが
全ての目的。

## 理想 (大きく)

- **確率論より決定論。** 「AI を信じて人間がレビューする」 から 「決定論ツールが証明
  する」 へ移した性質は、 リスクの *低減* でなく *除去*。 証明された結合律は黙って退行
  しないが、 確率論的な「正しそう」 は退行しうる。
- **人間のレビュー面を縮小する。** 理想は、「AI はこれを正しくやれたか?」 が 「この種の
  誤りは出荷され得ない」 に置き換わる領域を広げること。 レビューの労力は、 本当に
  形式化できない部分だけに集中できる。
- **二層ガードレールの物理層。** 方針層 (= *philosophy filter*) が 「機械検証可能 → AI に
  任せる / 不可 → 人間を残す」 を判断する。 AF はその判断を **コード書き込み時点で物理
  執行する**層 (= hook)。
- **本物の系譜を、 新しい的へ。** 「決定論的に検証できる領域を広げる」 は形式手法の
  50 年プログラム — 型システム・SMT ソルバー・CrossHair・Frama-C。 手法は Lindy に強い。
  時宜を得た新しい角度は、 それを AI が突如切実にした問いへ向けたこと: **どこまで
  *確認せず* に安全でいられるか?** コードが書かれる瞬間に引く信頼境界。
- **北極星。** AF がカバーする軸については、 AI への委任が人間の心配ゼロで済むまで島を
  広げる。

## 現実 (誠実に)

島は今日、 **薄い**。 hook を pass したコードについて、 AF が決定論的に認証するのは:

| 保証する軸 | tool | 誠実な注記 |
|---|---|---|
| Lint (PERF / SIM / FURB / ANN / F / RUF013) | ruff | 実在するが、 これは ruff — AF 固有ではない |
| データ移動 (中間 list / dict-keys-list / explicit-copy / loop 内文字列連結) | AF AST | 4 ルール、 AF 独自 contribution |
| 代数法則違反 (Monoid / Functor / …) | hypothesis | **名前認識した関数のみ** (= keyword 被覆 ~50%、 両側エラー) |
| 結合律 / 可換律の証明 | CrossHair (opt-in) | ルール照合でなく本物の **証明** の部分 — ただし狭い (binary 関数のみ) |

**島の外 — 依然 確率論、 依然 人間の仕事:** logic 正しさ・security・並行性・仕様/意図。
AF はこれらに触れない。 *構造* を検証し、 *意図* は決して検証しない。

機構自体の 2 つの誠実な性質:

- **hook は強制でなく助言。** *pass したものを認証* するが、 AI に pass するコードを
  *強制* はしない。 「このコードを exactly に書け」 と指示すれば、 hook を無視して維持
  する。 つまり *条件付き認証* (「pass したならこの軸は clean」) であって執行ではない。
- **実測価値は控えめで、 それも誠実に。** in-vivo A/B は — 旧 overclaim (+80%) を誤計測
  として撤回した後 — OFF 0/5 → ON 5/5 clean で、 有能なモデル上ではほぼ型注釈軸が主。
  [evidence_summary.ja.md](evidence_summary.ja.md) §1 と [limitations.ja.md](limitations.ja.md) 参照。

## 意図的に防ぐ危険

**過信が failure mode。** 「AF が pass = 安全」 は AF がカバーする軸についてのみ真。 人間が
AF pass を 「logic も security もレビュー済み」 と読めば、 安全装置が危険物になる — やって
いないレビューを省く免罪符になる。 だから島の海岸線は **常に大声で塗る**: 全ての主張は
適用範囲とセットで出す ([limitations.ja.md](limitations.ja.md) 「検証の正確な意味」 参照)。
境界が不明な保証は、 保証がないより悪い。

## 誠実な現在地

AF は **完成した大聖堂ではない。 本物の思想の、 最初の・薄い・誠実に計測された島**である。
その価値は島の面積に比例し、 面積を広げることが明示された program であって 後付けでは
ない。 有能なモデルが元々綺麗に書くことは、 これを損なわない: ガードレールが clean を
安価に確認し、 取りこぼしを捕捉する、 という まさに あるべき挙動を意味する。

理想は大きく — AI が書いたコードを、 ますます多くの軸で安全に委任できる世界。
現実は誠実に — 今日それは、 小さいが実在する、 境界を白日の下に引いた、 決定論的に
保証された一片である。
